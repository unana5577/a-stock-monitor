# A股智能看板 
 
> 只赚认知以内的钱。不追全市场，只盯关注的板块。3-5天左右小周期投资适用。 
 
--- 
 
## 这是什么？ 
 
一个 A 股统计看板，帮你回答三个问题： 
 
1. **今天大盘怎么样？** → 看量能/涨跌家数/指数结构，决定要不要加仓/减仓 
2. **关注的板块在什么阶段？** → 看生命周期 + 走势对比，知道什么时候进场、什么时候跑 
3. **最近发生了什么？** → 看新闻监控，了解宏观、地缘、板块动态 
 
**核心理念**：市场太大，不可能抓住所有机会。只做自己懂的板块，赚认知以内的钱。 
 
--- 
 
## 三大板块 
 
### 1. 资金流向监控 - "钱往哪流，哪就有机会" 
 
**功能**：监控交易日当天，指数/量能/涨跌家数，以及**你关注的板块**强弱变化，判断资金风险偏好 
 
**能告诉你**： 
- 关注的板块在走强还是走弱（相对走势） 
- 资金风格偏好（偏资源/偏成长的轮动信号） 
- 整体市场情绪（涨跌家数/恐慌） 
 
**怎么用**： 
- 大面积走弱 → 谨慎，控制仓位 
- 量能放大且走强 → 可以积极一点 
- 板块间轮动明显 → 跟着资金走，但别追高 
 
--- 
 
### 2. 板块生命周期 - "万物皆有周期" 
 
**功能**：追踪**你关注的板块**处于什么阶段，对比走势，参考历史 
 
#### 2.1 生命周期阶段 
 
| 阶段 | 特征 | 参考操作 | 
|------|------|----------| 
| 🌱 **启动** | 刚开始有动静，关注度低 | 可以研究，小仓位试水 | 
| 🚀 **爆发** | 资金涌入，涨势凶猛 | 已持有的爽，追高需谨慎 | 
| 📊 **分化** | 龙头继续涨，杂鱼开始掉队 | 去弱留强，关注龙头 | 
| 📉 **调整** | 下跌或横盘，资金撤离 | 观望，等下一个周期 | 
 
#### 2.2 T-60 走势归一化对比 
 
把多个关注板块的过去 60 天走势归一化，放在一起对比： 
 
- 一眼看出哪个板块在往上走、哪个在往下 
- 发现板块间的轮动规律 
- 避免被绝对价格误导 
 
#### 2.3 历史上涨周期监控 
 
记录每个板块近 180 天的上涨段/连涨表现： 
 
- 这一轮涨了多久？和历史相比处于什么位置？ 
- 哪些板块更容易走出“连续上涨段”？ 
- 帮你判断：行情还能持续，还是快到头了 
 
**怎么用**： 
- 优先关注「启动」和「爆发」期的板块 
- 用 T-60 对比确认趋势强度 
- 参考历史连涨段，判断还能走多远 
- 「调整」期就别去接飞刀了 
 
--- 
 
### 3. 新闻监控 - "信息就是金钱" 
 
**功能**：自动抓取和分类新闻，分为三个维度 
 
| 类型 | 内容 | 影响 | 
|------|------|------| 
| 🌍 **宏观** | 利率、GDP、政策会议 | 影响整体市场情绪 | 
| ⚔️ **地缘** | 国际关系、贸易摩擦 | 影响特定行业（芯片、能源等） | 
| 📈 **板块** | 行业政策、公司动态 | 直接影响相关板块 | 
 
**怎么用**： 
- 每天花 5 分钟扫一眼，保持信息同步 
- 重大新闻出现时，想想对持仓的影响 
- 利好不涨 → 可能有问题；利空不跌 → 可能见底

## 技术架构 
 
``` 
a-stock-monitor/ 
├── public/                 # 前端静态页（CDN 版 Vue/ECharts） 
├── server.js               # Node 服务：API + 静态托管 
├── fetch_*.py              # Python 数据抓取/计算（akshare 等） 
├── data/                   # 本地缓存/归档（含关注板块配置） 
└── runtime/                # 运行时缓存（本地文件） 
```

# 前端是 public/ 下的静态页面，由 server.js 自动托管

## 核心接口 
 
| 接口 | 说明 | 
|------|------| 
| `GET /api/snapshot/latest` | 今日看盘快照（指数/量能/涨跌家数，含 AI 可选） | 
| `GET /api/sector/lifecycle` | 板块生命周期概览（关注板块） | 
| `GET /api/sector/history` | T-60 走势归一化对比（关注板块历史数据） | 
| `GET /api/sector/rotation/sequence` | 轮动顺序与节奏（关注板块） | 
| `GET /api/news` | 新闻列表（支持日期筛选） |
