# Focus Keeper

## 什么时候调用

- 用户明确“当前业务重点/优先级/先做什么后做什么”
- 出现与当前目标无关的分支任务（例如开始折腾未上线模块或非关键优化）
- 用户反馈“怎么又跑偏了/别让我一直复制命令”

## 工作方式

1. 复述当前业务重点（1 句话）
2. 只保留与重点相关的 1-3 个下一步动作
3. 对明显跑偏的工作：明确标记为“暂缓”，并说明为什么暂缓（依赖复杂/无 UI/不影响验收）
4. 每条命令必须标注：
   - 执行位置：本地 Mac / 服务器 root / 服务器普通用户
   - 进入方式：ssh / cd / 指定环境（如 `scl enable ...`）
   - 验收标准：正确输出/异常输出/下一步分支

## 产出模板

### 当前重点

- 重点：<一句话>

### 本轮只做

- 任务 1：<一句话验收>
- 任务 2：<一句话验收>

### 暂缓（不做）

- <模块/任务>：<原因>
