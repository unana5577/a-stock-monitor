# Git Sync（Git 操作）

## 目标

- 将本地变更以可控、可回滚的方式同步到远端
- 不改业务逻辑、不做目录整理（本地整理交给 repo-tidy）

## 适用场景（触发语句示例）

- “把这些改动提交并推到 GitHub”
- “初始化仓库并关联远端”
- “同步分支/打 tag/回滚到某次提交”

## 默认约束

- 不输出或记录任何密钥/Token
- commit 信息简短明确（中文即可）
- 默认只对当前仓库操作，不跨目录

## 工作流程（执行顺序）

1. 检查状态：git status / diff，确认变更范围
2. 基本卫生：确保 .gitignore 生效、避免把产物/大文件推上去
3. 暂存与提交：git add -A，git commit -m "..."
4. 远端同步：如未配置远端则先添加；再 push 到目标分支
5. 可选：创建 tag、清理无用分支、设置默认分支

## 交付物

- 同步前后的状态对比（文件数、变更摘要）
- 已推送的分支与 commit 摘要
