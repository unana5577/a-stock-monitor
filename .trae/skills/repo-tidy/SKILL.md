# Repo Tidy（本地整理）

## 目标

- 让目录结构更清晰、页面文件更少更聚焦
- 明确“哪些应提交/哪些不应提交”
- 不做任何 commit/push（Git 操作交给 git-sync）

## 适用场景（触发语句示例）

- “上传 git 文件太多，帮我整理一下”
- “页面看起来很乱，收拾下目录”
- “把 dist/cache/log 这些不要进仓库”

## 默认约束

- 代码与界面输出中文、变更极简
- 优先通过 .gitignore 与移除已跟踪产物来降噪
- 需要保留的数据默认：只保留样例/模板，不提交大体量原始数据
- 不执行 commit/push；如需阶段性可执行 git add（仅暂存），并明确提示交接给 git-sync

## 工作流程（执行顺序）

1. 盘点仓库：列出“文件类型分布”和“异常大/明显产物目录”
2. 识别产物：dist、build、.cache、.next、.vite、coverage、logs、tmp、__pycache__、.DS_Store、*.log、*.csv/*.parquet（按项目约定）
3. 收敛忽略：创建/更新 .gitignore（只加必要条目，不堆砌）
4. 去跟踪产物：对已被 git 跟踪但不该提交的文件，执行移除跟踪（保留本地文件）
5. 归位结构：将零散文件移动到约定目录（如 src/ scripts/ docs/ data/），并更新引用路径
6. 最后可选：整理 README 的“入口/运行/目录说明”三段，减少噪声

## 交付物

- 一份“保留/忽略/移动”的变更清单
- 更新后的 .gitignore（如需要）
- 目录结构调整（如需要）
- 如发生引用路径变更，确保能正常运行（必要时执行 lint/typecheck）
