## 📅 日线数据规则

### 数据时间范围
- **起始日期**: 2025年5月19日
- **结束日期**: 今天
- **默认请求范围**: 1年（365天）

| 字段 | 说明 | 示例 |

### 日线更新规则
- **更新频率**: 每日
- **更新时间**: 收盘后 15:30
- **更新方式**: cron 定时任务

```bash
# 设置定时任务
crontab -e

# 添加以下行（每日 15:30 执行）
30 15 * * 1-5 cd ~/股票项目/backend && /usr/bin/python3 fetch_sector_data.py >> ~/股票项目/logs/update.log 2>&1
```

## 🔍 数据质量检查

### 检查项
- [ ] 数据完整性（无缺失日期）
- [ ] 数据准确性（价格范围合理）
- [ ] 数据一致性（不同源数据一致）
- [ ] 更新及时性（每日按时更新）

### 数据验证脚本

```python
import pandas as pd
import os

def validate_data():
    """验证数据质量"""
    data_dir = os.path.expanduser("~/股票项目/data")

    # 1. 检查指数数据
    index_dir = os.path.join(data_dir, "index")
    for file in os.listdir(index_dir):
        if file.endswith(".csv"):
            df = pd.read_csv(os.path.join(index_dir, file))
            print(f"✅ {file}: {len(df)} 条记录")

    # 2. 检查板块数据
    sector_dir = os.path.join(data_dir, "sector")
    for file in os.listdir(sector_dir):
        if file.endswith(".csv"):
            df = pd.read_csv(os.path.join(sector_dir, file))
            print(f"✅ {file}: {len(df)} 条记录")

validate_data()
```

## ⚠️ 常见问题

### Q1: 数据获取失败怎么办？
**A**: 检查网络连接和代理设置：
```bash
# 测试网络
ping 17.push2.eastmoney.com

# 如果使用代理，确保代理可用
export https_proxy=http://your-proxy:port
```

### Q2: 数据更新时间不对？
**A**: 检查定时任务是否正确：
```bash
# 查看定时任务
crontab -l

# 查看日志
tail -f ~/股票项目/logs/update.log
```

### Q3: 如何获取更多板块数据？
**A**: 修改 `sector_lifecycle_config.py` 中的板块列表：
```python
SECTORS = [
    "半导体",
    "云计算",
    "新能源",
    # 添加更多板块...
]
```
