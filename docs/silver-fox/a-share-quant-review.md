## ❌ 发现的数据问题

### 问题1：成交额占比单位不统一 ⚠️

**现象**:
- 指标数据中 `amount_share` 显示为原始数值（例如：2436569066.08）
- 判断逻辑中使用小数（例如：amount_high = 0.5）

**原因**:
```python
# sector_lifecycle.py 第130-132行
amount_share = sector_df["amount"].iloc[-1] / 10000  # 转换为万元
amount_share_history = (sector_df["amount"] / 10000).tolist()
amount_share_ma5 = calculate_amount_share_ma5(amount_share_history)
```

这里除以10000是转换为万元，但 `calculate_amount_share` 函数期望的是百分比：
```python
def calculate_amount_share(sector_amount: float, total_amount: float) -> float:
    return sector_amount / total_amount  # 返回 0-1 的百分比
```

**影响**:
- 所有板块都被判断为"高位区"（因为万元数值 >> 0.5）
- 位置判断完全失效
- 阶段判断也可能受影响

**解决方案**:
1. 在 `analyze_sector` 中计算总成交额
2. 将成交额转换为百分比后再计算
3. 或者在 `determine_position` 中同时支持两种单位

### 问题2：数据源返回 None ⚠️

**现象**:
```python
df = ak.stock_board_industry_hist_em(symbol='半导体', ...)
# 返回 TypeError: 'NoneType' object is not subscriptable
```

**原因**:
- 今天是2026-02-22（周日）
- 数据源可能不返回周末数据
- 或者数据源有其他问题

**影响**:
- 无法获取最新数据
- 功能无法正常运行

**解决方案**:
1. 检查数据源是否支持周末
2. 使用缓存数据
3. 增加数据源降级逻辑

### 问题3：显示名称不匹配 ⚠️

**现象**:
- 位置名称：补涨期
- 阶段信号：背离期B
- 显示名称：待观察

**原因**:
- 组合映射中没有 "补涨期 + 背离期B" 的组合
- 返回默认组合

**影响**:
- 用户看到的信号不明确
- 无法提供有效建议

**解决方案**:
1. 在 `POSITION_STAGE_COMBOS` 中添加缺失的组合
2. 或者调整判断逻辑，避免产生不匹配的组合
