# 板块生命周期判断系统设计文档

## 概述

为 A 股量化软件设计板块生命周期判断系统，通过 **位置 + 阶段** 两层结构，帮助用户识别板块当前的强弱状态和操作时机。

---

## 核心设计

### 一、两层架构

```
位置（宏观判断）+ 阶段（微观信号）= 完整的板块状态
```

**示例：**
- 位置 = 高位区 · 过热期
- 阶段 = 加速期
- 显示名称 = 过热·加速赶顶
- 操作建议 = 分批止盈

---

### 二、位置判断矩阵

#### 2.1 参数定义

| 参数 | 说明 | 计算方式 |
|------|------|----------|
| Alpha_5 | 5日相对大盘超额收益 | (板块5日涨幅 - 大盘5日涨幅) / 大盘5日涨幅 |
| Alpha_20 | 20日相对大盘超额收益 | (板块20日涨幅 - 大盘20日涨幅) / 大盘20日涨幅 |
| Amount_Share | 当日成交额占比 | 板块成交额 / 全市场成交额 |
| Amount_Share_MA5 | 5日平均成交额占比 | 近5日成交额占比均值 |

#### 2.2 位置矩阵

**基于 Alpha_20 + Amount_Share_MA5 组合判断位置：**

| 区域 | 位置名称 | Alpha_20 | Amount_Share_MA5 | 含义 |
|------|----------|----------|------------------|------|
| **高位区** | 过热期 | > +8% | > 50% | 极度拥挤，风险高 |
| | 强势期 | > +8% | 30-50% | 强势上涨，趋势健康 |
| | 背离期 | +3%~+8% | < 30% | 价格高位但资金撤退 |
| **中位区** | 热门期 | +3%~+8% | > 50% | 资金关注，热度上升 |
| | 活跃期 | +3%~+8% | 30-50% | 正常活跃，值得关注 |
| | 中性期 | -3%~+3% | 30-50% | 中性状态，等待信号 |
| | 磨底期 | -3%~+3% | < 30% | 底部磨底，资金潜伏 |
| **低位区** | 补涨期 | -3%~+3% | > 50% | 冷门板块突然放量 |
| | 冰点期 | < -3% | < 30% | 极度冷门，无人关注 |
| **异常区** | 异常期 | 高Alpha + 放量下跌 | 走势背离正常逻辑 |
| | 崩塌期 | 低Alpha + 放量暴跌 | 恐慌出逃 |

**阈值说明：**
- Alpha 阈值 ±3%/±8% 需要通过历史回测调优
- 成交额占比阈值 30%/50% 需要根据实际数据范围调整

#### 2.3 四大区域操作策略

| 区域 | 操作策略 | 原因 |
|------|----------|------|
| **异常区** | ❌ 不参与 | 走势异常，风险不可控 |
| **低位区** | ⚠️ 少关注，快进快出 | 涨幅有限，资金效率低 |
| **中位区** | ✅ 重点参与 | 主力建仓/启动，机会多 |
| **高位区** | ⚡ 谨慎参与，注意止盈 | 上涨空间有限，风险增加 |

### 三、阶段判断规则

#### 3.1 八个阶段定义

| 阶段 | 触发条件 | 含义 |
|------|----------|------|
| **衰退期** | 收盘价 < MA20 且 MA20 < MA60 | 趋势向下，均线空头排列 |
| **背离期(A)** | 收盘价 ≥ 近20日新高99.5%，但 RS变化20日 < 近20日最大值70% | 价格创新高但强度走弱 |
| **背离期(B)** | 成交额占比相对近20日高点回落 ≥ 20% | 资金开始撤退 |
| **加速期** | Bias_20 ≥ 8 且 MA20斜率 > 0 | 快速上涨，偏离均线过远 |
| **震荡期** | Bias_20 在 2~5 之间，且 MA20斜率绝对值 ≤ 0.2 | 横盘震荡，方向不明 |
| **启动期** | 收盘价 > MA60，MA60斜率 > 0，当日涨跌幅 ≥ 2%，成交额占比 ≥ 60% | 突破启动，资金进场 |
| **潜伏期** | 收盘价 < MA60，RS变化20日 ≥ -0.2，成交额占比 ≤ 30% | 底部潜伏，等待机会 |
| **兜底规则** | 不满足以上任何条件时 | 若收盘价 ≥ MA60 → 震荡期；否则 → 潜伏期 |

#### 3.2 阶段判断优先级

**按优先级从高到低判断：**

1. 衰退期（风险最高，优先识别）
2. 背离期(A) / 背离期(B)（警示信号）
3. 加速期（极端状态）
4. 启动期（机会信号）
5. 震荡期（中间状态）
6. 潜伏期（底部状态）
7. 兜底规则

#### 3.3 辅助参数定义

| 参数 | 说明 | 计算方式 |
|------|------|----------|
| MA20 | 20日均线 | 近20日收盘价均值 |
| MA60 | 60日均线 | 近60日收盘价均值 |
| Bias_20 | 20日乖离率 | (收盘价 - MA20) / MA20 × 100% |
| RS变化20日 | 相对强度变化 | Alpha_20 的变化率 |
| MA20斜率 | 均线方向 | MA20的线性回归斜率 |

### 六、技术实现要点

#### 6.1 数据源

- 板块数据：现有 `fetch_sector_data.py` 已有基础数据
- 大盘数据：上证指数作为基准
- 时间范围：近60个交易日

#### 6.2 计算逻辑

```python
# 伪代码
def analyze_sector(sector_data, market_data):
    # 1. 计算 Alpha
    alpha_5 = calculate_alpha(sector_data, market_data, days=5)
    alpha_20 = calculate_alpha(sector_data, market_data, days=20)

    # 2. 计算成交额占比
    amount_share = sector_data['amount'] / market_data['total_amount']
    amount_share_ma5 = calculate_ma(amount_share_history, days=5)

    # 3. 判断位置
    position = determine_position(alpha_20, amount_share_ma5)

    # 4. 计算辅助参数
    ma20 = calculate_ma(sector_data['close'], days=20)
    ma60 = calculate_ma(sector_data['close'], days=60)
    bias_20 = (sector_data['close'] - ma20) / ma20 * 100
    # ... 其他参数

    # 5. 判断阶段
    stage = determine_stage(sector_data, ma20, ma60, bias_20, ...)

    # 6. 组合名称
    display_name = combine_name(position, stage)

    # 7. 返回结果
    return {
        'position': position,
        'stage': stage,
        'display_name': display_name,
        'meaning': get_meaning(position, stage),
        'advice': get_advice(position, stage),
        'alpha_5': alpha_5,
        'alpha_20': alpha_20,
        'amount_share': amount_share,
        'amount_share_ma5': amount_share_ma5
    }
```

### 七、调优计划

#### 7.1 阈值调优

| 参数 | 初始值 | 调优方法 |
|------|--------|----------|
| Alpha 阈值 | ±3%/±8% | 历史回测，看各阶段预测准确率 |
| 成交额占比阈值 | 30%/50% | 根据实际数据分布调整 |
| Bias 阈值 | 2/5/8 | 回测验证 |
| 成交额占比回落阈值 | 20% | 回测验证 |

#### 7.2 回测指标

- **阶段预测准确率**：启动期后5日涨幅为正的比例
- **风险预警准确率**：衰退期/背离期后5日下跌的比例
- **信号频率**：各阶段出现的频率，避免某个阶段过于频繁
