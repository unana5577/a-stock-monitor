## 数据质量检查

### 检查项
- [ ] 数据完整性（无缺失日期）
- [ ] 数据准确性（价格范围合理）
- [ ] 数据一致性（不同源数据一致）
- [ ] 更新及时性（每日按时更新）

### 异常处理
- 数据缺失 → 记录日志，标记异常
- 数据异常 → 发送告警
- 接口失败 → 重试3次，仍失败则告警

## API 接口设计

### 1. 获取板块数据
```
GET /api/sector/:sectorName
参数：
- startDate: 起始日期（可选）
- endDate: 结束日期（可选）
返回：板块历史数据
```

### 2. 获取指数数据
```
GET /api/index/:indexCode
参数：
- startDate: 起始日期（可选）
- endDate: 结束日期（可选）
- interval: 日线/分时（可选）
返回：指数历史数据
```

### 3. 获取市场情绪
```
GET /api/market/breath
参数：
- date: 日期（可选，默认最新）
返回：涨跌家数
```

### 4. 获取新闻
```
GET /api/news
参数：
- date: 日期（可选，默认最新）
- sector: 板块（可选）
- sentiment: 情绪（可选：利好/利空）
返回：新闻列表
```
